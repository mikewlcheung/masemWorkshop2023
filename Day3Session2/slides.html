Examples and applications

Example 3: A mediation model

  • Schutte, Keng, and Cheung (2021)1 examined the association among dispositional mindfulness, gratitude, and emotional intelligence via a mediation model with 41 samples.
  • Although we may individually meta-analyze the bivariate relationships between EI and mindfulness, EI and gratitude, and mindfulness and gratitude, these results cannot tell us the psychological mechanism. Researchers want to address the mediating role of EI.
  • Another feature of this study is that there is no study including all three variables. Therefore, this study can address research questions beyond primary studies. However, one limitation is that there may be a large degree of heterogeneity. Future studies may test the validity of the mediation model by including all three variables.

Schutte, Keng, and Cheung (2021), Figure 1

  • An interesting feature of this dataset is that each study only examines the bivariate relationship. MASEM allows testing theories beyond the primary studies bivariate relationship.

A function to convert rows into a 3x3 correlation matrix

# |             | Mindfulness | EI       | Gratitude |
# |-------------+-------------+----------+-----------|
# | Mindfulness | 1.0         |          |           |
# | EI          | mat[2,1]    | 1.0      |           |
# | Gratitude   | mat[3,1]    | mat[3,2] |       1.0 |

create.matrix <- function(x, type=c("mindfulness and emotional intelligence", 
                                    "mindfulness and gratitude", 
                                    "emotional intelligence and gratitude")) {
  mat <- matrix(NA, ncol=3, nrow=3)
  diag(mat) <- 1
  type <- match(type, 
                c("mindfulness and emotional intelligence", 
                  "mindfulness and gratitude", 
                  "emotional intelligence and gratitude"))
  switch(type,
         "mindfulness and emotional intelligence" = mat[1, 2] <- mat[2, 1] <- unlist(x),
         "mindfulness and gratitude" = mat[1, 3] <- mat[3, 1] <- unlist(x),
         "emotional intelligence and gratitude" = mat[2, 3] <- mat[3, 2] <- unlist(x))
  mat
}

Testing the direct effect equals the indirect effect

  • The previous analysis suggests that the magnitude of the direct effect is comparable to that of the indirect effect. We may formally run a test to verify it.
  • We add a nonlinear constraint c == a*b to test the above hypothesis
  • The LR test is \(\chi^2(df=1) = 0.005, p=.94\), suggesting that there is no evidence to reject the hypothesis that the direct effect equals the indirect effect.
## Model with direct effect equals indirect effect
model2 <- "Gratitude ~ c*Mindfulness + b*EI
           EI ~ a*Mindfulness
           Mindfulness ~~ 1*Mindfulness
           ## Define direct and indirect effects
           Direct := c
           Indirect := a*b
           ## Add a constraint to ensure c = a*b
           c == a*b        
"

Parameter-based MASEM

  • The approaches covered so far are known as correlation-based MASEMcorrelation matrices are synthesized first, then structural models are fitted.
  • An alternative is called parameter-based MASEM (Cheung & cheung, 2016).2:
    • It fits structural equation models and gets each studys parameter estimates and their sampling covariance matrices.
    • A multivariate meta-analysis is conducted using parameter estimates as effect sizes.
  • Advantage: random effects are on the parameters, e.g., indirect and direct effects.
  • Disadvantages:
    • Complete data are required.
    • No model fit indicated on the proposed model.
model <- "Beh ~ b*Int + c*PB
          Int ~ a*PB
          PB ~~ 1*PB"

Tips in MASEM

  • This session will discuss some common issues and tips in MASEM.

It takes a long time to run my models!

  • We may speed up the analyses by using more cores available in your computers.
library(metaSEM)

## Show how many cores are in my computer
parallel::detectCores()
## [1] 8
## Use all cores-2, i.e., keep 2 cores for other operations.
mxOption(key='Number of Threads', value=parallel::detectCores()-2)

Why are my data strings not numeric?

  • There may be white space characters in Excel if we accidentally add spaces before or after the numbers. These variables will be read as strings instead of numbers. We may use as.numeric() function to convert them to numbers.
## x is a string vector
x <- c(10.1, 23.4, "2.1 ", " 3.4", "\t4.2", "3.1\n")
x
## [1] "10.1"  "23.4"  "2.1 "  " 3.4"  "\t4.2" "3.1\n"
is.numeric(x)
## [1] FALSE
is.character(x)
## [1] TRUE
## Convert it to numeric
y <- as.numeric(x)
y
## [1] 10.1 23.4  2.1  3.4  4.2  3.1
is.numeric(y)
## [1] TRUE

How do I know if my correlation matrices are proper?

  • There are restrictions on correlation matrices, e.g., they cannot be larger than 1, which is known as positive definite. The results may be problematic when some correlation matrices are non-positive definite.
  • We may check the definite of the matrices with is.pd() function.
## X1 is okay
X1 <- matrix(c(1, 0.8, 0.8, 1), nrow=2)
X1
##      [,1] [,2]
## [1,]  1.0  0.8
## [2,]  0.8  1.0
is.pd(X1)
## [1] TRUE
## X2 is not okay, as the correlation is 1.2.
X2 <- matrix(c(1, 1.2, 1.2, 1), nrow=2)
X2
##      [,1] [,2]
## [1,]  1.0  1.2
## [2,]  1.2  1.0
is.pd(X2)
## [1] FALSE
## X3 is not okay, as the correlation matrix is not symmetric.
X3 <- matrix(c(1, 0.6, 0.8, 1), nrow=2)
X3
##      [,1] [,2]
## [1,]  1.0  0.8
## [2,]  0.6  1.0
isSymmetric(X3)
## [1] FALSE

What should I do when the solution is non-convergent?

  • Statistical packages use iterative procedures to estimate the parameters. When the iterations are convergent (code 0 or 1 in OpenMx), we can trust the results; otherwise, we should not interpret the results.
  • The results are okay:
OpenMx status1: 0 ("0" or "1": The optimization is considered fine.
Other values indicate problems.)
  • The results are not okay:
OpenMx status1: 5 ("0" or "1": The optimization is considered fine.
Other values may indicate problems.)
Warning message:
In print.summary.meta(x) :
  OpenMx status1 is neither 0 or 1. You are advised to 'rerun' it again.

## Suppose the results are stored in an object called "fit"
## We may improve the fit by rerunning it with rerun().
## The default extraTries is 10.
fit <- rerun(fit, extraTries=50)
summary(fit)
  • A common reason for non-convergent solutions is that there is no data or not enough data. We may use the following code to check it.
  • There are several options to deal with this issue:
    • Try a fixed-effects model.
    • Simplify the model by using fewer variables (see below).
